# 2026-06-28 실험 — HAETAE 결함주입 vs 대응기법 (대학생용 설명)

> 더 자세한 수식·표는 `attack_defense_analysis.md`, 정식 리포트는 `fullsign_keyrecovery_report.md`.
> 이 문서는 학부 수준에서 **무엇을·왜·어떻게** 했는지 정확한 용어로 정리한다.

---

## 1. 배경

- **격자 기반 Fiat–Shamir 서명**: 양자내성암호(PQC) 전자서명의 대표 방식. 비밀키 `s₁`,
  공개 챌린지 `c`, 마스킹 난수 `y` 로부터 응답 `z`를 만들고, 거부 샘플링으로 `z`가 비밀키를
  누설하지 않게 한다. Dilithium, **HAETAE** 가 이 계열이다.
- **HAETAE**: 한국형 PQC(KpqC) 서명 후보. bimodal 분포 + 하이퍼볼 샘플링으로 서명 크기를 줄였다.
- **문제의식**: 알고리즘이 수학적으로 안전해도, **임베디드 칩 구현**에서는 물리적 공격
  (결함주입, FIA)에 취약할 수 있다. 본 실험은 그 취약성과 대응을 정량 비교한다.

**핵심 응답식:**
```
z = y + (−1)^b · c · s₁        (R_q = Z_q[x]/(x^256 + 1),  q = 64513)
```
`y`=마스킹 난수(비밀, 매 서명 갱신, 고정소수점 스케일 Lₙ=8192), `c`=챌린지(공개, ±1, 무게 τ),
`s₁`=비밀키(표적), `b`=bimodal 부호.

---

## 2. 위협 모델 (결함주입)

- 공격자는 칩에 **클럭/전압 글리치**로 **단일 결함**을 주입해, 특정 연산을 **스킵**하거나 값을
  **변조**할 수 있다.
- 본 실험의 결함모델 6종: `skip`(명령 생략), `zeroing`(0으로), `bitflip`, `byteflip`,
  `set_const`(상수 고정), `loop_abort`(루프 조기종료).
- 관찰: 공격자는 방출된 서명과 공개 정보(`c`, 공개키)만 본다. `y`·`s₁`은 직접 못 본다.

---

## 3. 왜 비밀키가 복구되는가 (복구 원리)

응답 `z`는 비밀항 `c·s₁` 을 **마스킹 `y`** 로 가린 형태다. 공격의 본질은 **`y`를 제거하거나
예측**하는 것이다. `y`만 사라지면 `z = (−1)^b c·s₁` 이 되고, `c`는 공개이므로 NTT 영역에서
다음으로 비밀키를 **직접 역산**한다(공개값만 사용):

```
ŝ₁[k] = NTT(z / Lₙ)[k] · ĉ[k]⁻¹  (mod q),   s₁ = InvNTT(ŝ₁),   ĉ = NTT(c)
```

이 복구식은 HAETAE 레퍼런스 정수산술과 **비트 단위 768/768 일치**로 검증되었다.

### 공격점별 요약 (복구 방법 / 필요한 서명 수)

| 서명 라인 | 결함 효과 | 방출 z | 복구 방법 | 트레이스 |
|---|---|---|---|---|
| **add_y (T2)** | +y 스킵 (y는 유지) | `z = Lₙ·c·s₁` | 위 역산식 직접 | **1 (단일)** |
| y_sample | y=0 | `z = c·s₁` | 위와 동일 | 1 |
| seed_gen | 시드 고정 → y 예측 | `z = y(예측)+c·s₁` | `s₁=NTT((z−y)/Lₙ)·ĉ⁻¹` | 1 |
| cs1_mul (T1) | c·s₁ 스킵 | `z ≈ y` (nonce 노출) | 정상 z_n과 `c·s₁=z_n−z` | 2 |
| sign_bit | b 반전 | `z = y−(−1)^b c·s₁` | 차분 `z_n−z=2c·s₁` | 2 |
| unpackA/commit/lsb/challenge | c 경로 오염 | c′≠c 무효서명 | 정상·오류 차분(Lee 등) | 2+ |
| reject_B1 (RB) | 거부판정 스킵 | 경계초과 z | 다수의 통계/격자 | 다수 |
| key_mem | 비밀키 변조 | 변조키 서명 | 차분 오류분석(DFA) | 2+ |

- **가장 치명적 = T2**: 단일 결함·단일 서명으로 `s₁` 전체가 즉시 복구된다. 본 연구 주표적.
- 트레이스 수가 적을수록 위험하다(1 ≫ 2 ≫ 다수).
- (참고) 본 시뮬은 "결함이 의도 명령에 들어갔다"는 가정의 논리 결과다. 글리치가 그 명령에
  *떨어질 확률*은 별개이며, STM32F3 실측에서 T2 단일트레이스 복구는 0.051%(32/62,720) 빈도였다.

## 4. 대응기법 — 적용 위치와 동작

### ① HAETAE-IRV (제안)
**적용 위치:** 응답 `z` 확정(거부 통과) 직후, 힌트/인코딩 이전.
- **M1 (재계산 검증):** `c·s₁`을 독립 재계산해 `z′=y+(−1)^b c·s₁` 과 방출 `z`를 비교
  → **연산 경로 결함(T1, T2)** 탐지.
- **M2 (서명경계 재검사):** `‖z‖²`을 독립 계산해 **생성경계 B₁**과 비교 → **거부 우회(RB)** 탐지.
- **M3 (키 체크섬):** `s₁` 무결성 검사 → **영속 키변조(key_mem)** 탐지.
- **M4 (무분기 감염):** 잔차 δ를 PRF로 출력에 분기 없이 확산 → δ≠0이면 서명 무효화.
  "if 탐지 then 중단" 분기가 없어 **검사-스킵형 2차 결함**에 강함.
- 특성: **연산 무결성**(z=y+(−1)^b c·s₁ 관계 성립 여부)을 검증. 입력 `y`·`c`의 *품질*은 미검증.

### ② Lee 등 (JKIISC 2026)
**적용 위치:** 시드 생성부 / 부호·난수 샘플부 / 서명 종료부.
- 정상성 검사(XOF 버퍼 퇴화) + 시드 독립 재유도 + 부호·하이퍼볼 샘플 재계산 + 서명후검증(Verify).
- 특성: **난수 생성 경로·챌린지 경로**를 넓게 검증. 단 Verify가 **검증경계 B₂**(>B₁)를 써서
  거부우회를 놓침.

### ③ 전체 이중연산 (Double)
**적용 위치:** 서명 전체 래퍼. 2회 수행 후 출력 비교, 불일치 폐기.
- 특성: 일시적 결함에 강함. 단 **영속 결함**(key_mem)은 두 실행 모두 동일해 미탐.

|---|:--:|:--:|:--:|:--:|
| add_y (T2) — 단일트레이스 직접복구 | ○ | **●** M1 | ● | ● |
| cs1_mul (T1) | ○ | **●** M1 | ● | ● |

| | 무방어 | IRV | Lee 등 | 이중연산 |

## 6. 방지율 해석 — 높을수록 좋다, 단 단일값만 보면 안 된다

- **정의:** `방지율 = (막은 키복구 결함) / (키복구 유발 결함, 47)`. **높을수록 안전.**
- **주의:** 분모 47건은 **심각도(단일/차분/통계)** 와 **방어 영역(연산 무결성 vs 난수 품질)** 이
  다른 결함을 섞는다. 따라서 단일 전체값은 오해를 부른다.
- IRV의 전체 **21.3%** 는 "성능 부족"이 아니라, **IRV 설계 범위 밖(난수경로 37건)** 이 분모에
  포함된 결과다. **공정하게는 class별로 봐야 한다.**

### class별 방지율

| 결함 class | 건수 | **IRV** | Lee 등 | 이중연산 |
|---|---:|---:|---:|---:|
| **응답무결성·직접 키복구** (T1·T2·RB·key_mem) | 10 | **100%** | 90% | 70% |
| 난수·challenge 경로 (seed·y·sign_bit·unpackA·commit·lsb·challenge) | 37 | 0% | 100% | 100% |
| 전체 | 47 | 21.3% | 97.9% | 93.6% |

- **IRV:** 표적 class(직접 키복구·거부우회)에서 **100%**. 여기엔 (i) 가장 치명적인 **단일트레이스
  직접복구(T2)**, (ii) 다른 두 기법이 못 막는 **거부우회(RB)** 포함. **무분기·최저비용(≈1.0×).**
- **Lee 등:** 난수경로 100%, 응답무결성 90%(RB 미방지), **≈1.5×.**
- **이중연산:** 응답무결성 70%(영속 key 미방지), **≈2×.**

**정리:** 방지율은 높을수록 좋고, **"class별 방지율 + 오버헤드"** 로 평가하는 것이 공정하다.
IRV는 *가장 치명적인 직접 키복구·거부우회를 최저비용·무분기로 100% 차단* 하는 데 특화된 대안이며,
Lee 등은 *난수경로를 넓게 막는* 대안이다(본 연구는 결합하지 않고 각각 비교).

## 7. 용어 정의

| 용어 | 정의 |
|---|---|
| **결함주입공격(FIA)** | 클럭/전압 등으로 연산 중 칩 상태를 교란해 비밀을 추출하는 능동 공격 |
| **마스킹 난수 y (nonce)** | 응답에서 비밀항을 가리는, 매 서명 새로 뽑는 비밀 난수 |
| **방지율** | 키복구 유발 결함 중 대응기법이 막은 비율(높을수록 좋음) |
| **트레이스** | 키복구에 필요한 (정상/오류) 서명 개수 (1=단일, 2=차분, 다수=통계) |
| **T2 (add_y 스킵)** | +y를 건너뛰어 z=c·s₁ 을 만드는 단일트레이스 직접복구 공격(최고 위험) |
| **RB (거부 우회)** | 거부 샘플링(‖z‖≥B₁ 재시작)을 스킵해 경계초과 z를 방출시키는 공격 |
| **B₁ / B₂** | 서명 생성 경계 / 검증 경계. **B₁ < B₂**(9838.98 < 12777.52)이라 RB가 검증을 통과 |
| **영속 키변조(key_mem)** | 비밀키가 메모리에서 지속 변조된 상태(이중연산이 미탐) |
| **DFA** | 차분 오류분석 — 정상·오류 출력의 차이로 비밀을 복구 |
| **HAETAE-IRV** | 제안 대응. z 재계산(M1)+경계 재검사(M2)+키 체크섬(M3)+무분기 감염(M4) |
