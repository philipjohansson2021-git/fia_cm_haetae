# 2026-06-28 실험/시뮬레이션 요약

## 수행 내용
**실제 HAETAE 전체 서명**(공식 레퍼런스, x86)에 서명 알고리즘 **라인(연산)별 결함주입**을
수행하고, 대응기법이 **실제 키복구**를 막는지 비교했다. **IRV vs Lee 등(JKIISC'26)을
별개**로 비교(결합하지 않음), 참고로 전체 이중연산 포함.

## 방법 (요지)
- 계측 서명 `sign_fi.c`: 레퍼런스 sign.c를 패치해 12개 라인에 결함 훅 + baseline/IRV/Lee-Ha 내장.
- 결함모델 6종: skip, zeroing, bitflip, byteflip, set_const, loop_abort.
- 더블체크: 무결함 IRV==정상(오탐0), T1/T2/RB/key 결함 탐지 등 **불변식 9/9 PASS**.
- 우선 지표 = **실제 키복구 여부**(복구 가능성보다). hint·무효과·크래시 제외. T2 단일트레이스
  직접복원은 별도 검증(레퍼런스 산술 768/768 일치).
- 커버리지는 아키텍처 독립(x86=ARM); 절대 오버헤드만 STM32F4 실측 예정.

## 핵심 결과 (키복구 유발 47건 중 방지율)
| 대응기법 | 키복구 방지 | 상대 오버헤드(x86) | 미방지 |
|---|---:|---:|---|
| **HAETAE-IRV** | 10/47 = 21.3% | **≈1.0×** | 난수경로(seed/sign_bit/y)·challenge |
| Lee 등(JKIISC'26) | 46/47 = 97.9% | ≈1.5× | **거부우회(RB)** |
| 전체 이중연산 | 44/47 = 93.6% | ≈1.7× | 영속 키메모리 |

- **IRV는 응답무결성·직접키복구 class(T1·T2·RB·key) 10/10 전담** — 단일트레이스 직접복원(T2)과
  다른 기법이 못 막는 **거부우회(RB)** 포함, **최저비용·무분기**.
- **Lee–Ha는 난수경로 광범위(97.9%)** 하나 **RB 미방지·고비용**. 이중연산은 영속 키결함 미방지·2배.
- IRV와 Lee–Ha는 **서로 다른 결함 class를 겨냥하는 대안**(우열 단순비교 부적절).
