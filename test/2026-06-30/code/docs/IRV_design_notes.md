# HAETAE-IRV 보강판 설계 정리 (haetae_sign_cm.c, `-DHAETAE_VARIANT_IRV`)

> 단일 오류주입에 대해 서명 응답 `z = y + (−1)^b c·s1` 및 그 상·하류 연산의 무결성을
> 검증하고, 모든 잔차를 단일 누산기 `cm_delta`(δ)에 OR로 모은 뒤 **무분기 감염형 마스킹**으로
> 출력을 무효화한다. δ=0이면 서명 불변(오탐 0), δ≠0이면 서명 난수화(키유출 0).

## 메커니즘 6종 (실행 순서대로)

| ID | 위치(코드) | 검사 내용 | 주로 잡는 결함 | 무분기 |
|---|---|---|---|:--:|
| **M3** 키 체크섬 | 서명 시작 전 | unpack한 `s1`의 FNV-1a 체크섬 vs 첫 무결함 기준 | 영속 키 변조 | ✔ |
| **M1** c·s 재계산 | z 확정 직후 | `c·s1`을 독립 재계산 → `z1 == y1+(−1)^b c·s1'` 를 계수별 XOR 누적 | **T1**(c·s 스킵)·**T2**(+y 스킵)·loop-abort·데이터변조 | ✔ |
| **M2** 노름 재검사 | z 확정 직후 | `‖z‖`가 **서명경계 B1 _및_ 바이모달 보조경계 B0** 를 만족하는지 재검사 | **RB**(거부 샘플링 우회) | ✔ |
| **M1b** 시드 정상성 | z 확정 직후 | 샘플링 시드 버퍼가 all-zero / all-equal 패턴인지 | **SEED**(시드 squeeze 스킵→0 고정) | ✔ |
| **M1c** 부호/논스 재유도 | z 확정 직후 | 같은 시드·카운터로 (y,b) 재샘플 → `bb==b && y==yb` | **SIGNBIT**(b 반전; 부호뒤집힌 서명도 유효해 검증/곱셈재계산으론 미탐) | ✔ |
| **M1′** 검증 등가 | pack_sig 후 | 방출 서명을 표준 검증식(`crypto_sign_verify_internal`, vk=sk 선두)으로 재구성·대조 | **LSB·UNPACK** (자기일관적이나 잘못된 서명) | ✘(내부 분기) |
| **M4** 감염 마스킹 | 마지막 | `factor = (δ≠0)?0xFF:0`; `sig ^= SHAKE256(비밀시드‖δ) & factor` | (탐지된 모든 결함 무효화) | ✔ |

→ M1·M2·M1b·M3 의 핵심 검사는 **무분기**(XOR/OR 누적)라 "검사 분기 스킵" 2차 결함에 강하다.
   M1′(표준검증)만 내부 분기가 있으나, 이는 *상류 결함 보강용*이고 핵심 표적(T1/T2/RB)은
   무분기 경로(M1/M2)가 이미 포착한다.

## 결함지점 → 담당 메커니즘

| 결함지점 | 슈도 라인 | 담당 |
|---|---|---|
| UNPACK (A 변조) | 1 | M1′ |
| SEED (시드 스킵) | 2 | **M1b** |
| SIGNBIT (b 반전) | 3 | **M1c** |
| LSB (poly_lsb 스킵) | 5 | M1′ |
| CS = T1 (c·s 스킵) | 7 | M1 |
| ADDY = T2 (+y 스킵) | 8 | M1 |
| REJECT = RB (거부판정 스킵) | 9 | **M2(B0+B1)** |
| 영속 키 변조 | — | M3 |

## 이전 버전 → 보강판 변경점

| 항목 | 이전 | 보강 | 이유/효과 |
|---|---|---|---|
| **M2 노름** | **B1만** 재검사 | **B1 + B0** 재검사 | 이전엔 B0(바이모달)로 거부될 RB-z를 놓침 → baseline과 동일 출력(유출). B0 추가로 **RB 완전 차단**. (설계서 Algorithm 6의 "B1 및 보조경계 B0" 의도 복원.) |
| **시드 정상성** | 없음 | **M1b 신규** | SEED 결함(seedbuf 0 고정) 미탐 → 이제 차단. |
| **상류 결함** | 없음 | **M1′ 검증 폴드 신규** | M1은 z=y+c·s 일관성만 봐서 c·y·b가 *일관되게* 변조된 LSB/UNPACK/SIGNBIT를 못 잡음(재계산도 같은 변조값 사용). 표준검증으로 "유효하지 않은 서명"을 포착해 차단. |
| **커버리지** | CS·ADDY만 (2/7) | **7지점 전면** (RB 포함) | leeha는 RB를 구조적으로 놓침(검증경계 B2 > 서명경계 B1) → **IRV만 RB 차단** = 핵심 차별점. |
| **비용** | ~+2% (좁은 M1만) | ~ leeha 수준(+5~8%, 검증 폴드 포함) | 전면 커버리지의 대가. 단 여전히 double(≈2×) 대비 절반 이하. (STEP 7/8 실측으로 확정.) |

## 실험에서의 위치(검증 방법)
- 축 B(SW 라인주입)에서 7결함 × 4변형 순회 → 방출 다이제스트 분류로 커버리지 측정.
- 보강 전: irv가 SEED/SIGNBIT/UNPACK/LSB/REJECT 에서 'LEAK'.
- 보강 후 기대: irv 7지점 모두 'blocked', leeha만 REJECT 'LEAK' → **IRV 전면 우위**.
- 정상서명(FL_NONE)은 δ=0 보장 → 4변형 모두 GOLDEN 동일(오탐 0).
